# 🤖 DL → Excel Web-Based App 設計規格書 (Design Spec)

本文件規劃將原有的 Python (Tkinter) `DL_to_Excel` 工具，完全移植為一個**純前端、免伺服器、百分之百本機離線運作**的極簡網頁版應用程式 (HTML/JS/CSS)。

---

## 🎯 專案定位與 UI 風格
1. **隱私與安全優先 (Fail-Safe)**：所有資料皆在本機瀏覽器內解析並生成 Excel，絕不經過網路傳輸，100% 避免半導體敏感資料洩露的風險。
2. **瑞士極簡主義 (Swiss Minimalism)**：大量留白、嚴謹網格對齊、清晰的無襯線字體、不對稱排版，消除視覺雜訊。
3. **新興科技感 (Emerging Tech)**：以暗色調背景搭配細緻漸層（靛藍到紫色）、精確的微動畫（Micro-animations）與即時狀態列，呈現高級的產品體驗。

## ⚙️ 核心業務邏輯與移植對照

### 1. Datalog 解析器 (Parser)
與原 Python 邏輯 100% 對齊：
*   **略過前 5 行**：略過 metadata。
*   **欄位名稱解析 (第 6 行)**：
    *   使用 JS 正則表達式 `/[ /<>]+/` 切割整行文字。
    *   若第一欄為空字串，自動將其移除。
    *   **欄位去重**：若遇到重複的欄位名稱（例如 `VAL` 出現兩次），自動重命名為 `VAL.1`、`VAL.2`，避免資料覆寫。
*   **資料列解析 (第 7 行起)**：使用同正則表達式分割，若行資料無效 (Bad lines) 則自動跳過 (on_bad_lines='skip')。
*   **數值型態判定 (Numeric Auto-convert)**：
    *   對每一欄進行檢查，如果**整欄的所有資料**皆能成功轉換為數字，則將該欄所有值轉為 Number 型態；否則保留原始字串，以防損毀科學記號或十六進位資料。

### 2. 多檔案合併 (Merge) 的防呆機制 (Fail-Safe Check)
*   **結構一致性檢查**：多檔合併時，系統會以第一個檔案的欄位（Columns）為基準。逐一比對後續檔案的欄位。
*   **錯誤阻斷**：若發現有欄位數量不符、少欄或多欄的情況，系統會**立即中斷轉換**，並在狀態列以紅色醒目字體顯示詳細錯誤訊息（例如：「檔案 B 多了 XXX 欄，合併中斷」），不進行任何寫入，確保匯出資料的正確性。

### 3. 下載與輸出模式 (Export Modes)
*   **合併模式 (Merge)**：
    *   將所有檔案的資料合併為單一表單。
    *   Column A 強制插入 `Source` 欄位，記錄其原始檔名（不含副檔名）。
    *   彈出欄位選擇選單（Modal），`Source` 欄位強制選取且 disabled；使用者可搜尋與多選其他欄位。
    *   匯出為 `datalog_merged.xlsx`。
*   **分開儲存並打包模式 (Split)**：
    *   彈出欄位選擇選單，使用者選擇要匯出的欄位。
    *   逐一檔案進行欄位過濾與解析，生成各自的 Excel 檔案。
    *   使用 `JSZip` 將所有個別的 Excel 檔案（如 `file1.xlsx`, `file2.xlsx`）壓縮打包。
    *   最後在前端觸發下載單一 `.zip` 壓縮檔，避免多次點擊與瀏覽器多檔案下載攔截。

## 🛠️ 測試計畫 (Testing Plan)
1. **單檔與多檔基本轉換測試**：確保匯出之 Excel 檔案可被 Microsoft Excel / LibreOffice 正常開啟。
2. **極大檔案解析效能測試**：在前端測試 10,000+ 行資料的解析速度與記憶體佔用。
3. **防呆欄位不一致測試**：故意上傳欄位不同的 Datalog 檔案，確認系統能精確阻斷並報錯。
4. **離線環境測試**：關閉網路連線，確認網頁仍能透過本地 `libs/` 內的 JS 檔案正常運行。
